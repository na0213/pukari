# CLAUDE.md — Pukari開発ルール

## プロジェクト概要

- **アプリ名**: Pukari
- **種別**: PWA（Progressive Web App）
- **技術スタック**: React + TypeScript + Supabase
- **対象ユーザー**: AuDHD当事者を中心とした、思考の散らかりやすい人
- **コア思想**: 「タスク管理」ではなく「自分を責めないための場所」
- **認証方針**: v0.1は匿名認証（ログイン画面なし）。v0.3以降で「あとからアカウント作成」（匿名→正規ユーザー昇格）を追加予定

## ファイル・コマンド操作の安全ルール

### 1. 既存ファイルの上書き禁止（確認必須）

- 既存ファイルを編集・上書きする前に、必ず以下のように確認する
  > 「〇〇を上書きしますが、よろしいですか？」
- 確認なしに既存ファイルの内容を変更しない
- 可能であれば変更前のバックアップを作成する（例：`filename.bak`）

### 2. 削除コマンドの実行禁止

- `rm`、`del`、`rmdir` などの削除系コマンドは原則として実行しない
- どうしても必要な場合は、対象ファイル名と理由を明示して承認を得てから実行する
- **`rm -rf` のような再帰的強制削除は、いかなる場合も実行しない**

### 3. パッケージ追加は事前説明と承認が必要

`npm install`、`pip install`、`brew install` などのパッケージ追加コマンドは、実行前に以下を説明する：

1. **何をインストールするか**（パッケージ名）
2. **なぜ必要か**（目的・用途）
3. **影響範囲**（グローバルかローカルか）

説明後、承認を得てから実行する。

### 4. 不明なコマンドは実行前に日本語で説明

私がエンジニアではないことを常に意識する。

実行しようとするコマンドが技術的・専門的な場合は、実行前に日本語で以下を説明する：

1. **このコマンドは何をするか**（平易な言葉で）
2. **実行するとどうなるか**（結果・影響）
3. **リスクがある場合はその内容**

説明後、「実行してよいですか？」と確認する。

## コーディング規約

### 言語・フレームワーク
- TypeScript を使用（JavaScript は使わない）
- React 18+ with Hooks
- CSS Modules または Tailwind CSS（要初期決定）
- Supabase Client SDK

### ファイル構成
```
src/
  components/    # UIコンポーネント
  hooks/         # カスタムフック
  lib/           # Supabaseクライアント、ユーティリティ
  pages/         # ページコンポーネント
  styles/        # グローバルスタイル、CSS変数
  types/         # TypeScript型定義
public/
  manifest.json  # PWA マニフェスト
  sw.js          # Service Worker
```

### 命名規則
- コンポーネント: PascalCase（`BubbleView.tsx`）
- フック: camelCase with `use` prefix（`useBubbles.ts`）
- 型定義: PascalCase with suffix（`BubbleState`, `SkyColor`）
- CSS クラス: kebab-case（`bubble-container`）

### アクセシビリティ
- 感覚過敏への配慮: アニメーションは `prefers-reduced-motion` を必ず尊重
- 色覚配慮: 色だけに依存しない状態表現
- スクリーンリーダー: 適切な `aria-label` を付与

### パフォーマンス
- WebGL は v0.1 では使わない（CSS アニメーションのみ）
- 画像の遅延読み込み
- Service Worker によるオフライン対応

## フェーズ完了時のルール

各フェーズ（Phase）の作業が終わったら、以下の手順を必ず守る：

1. `npm run dev` で開発サーバーを起動して動作確認を行う
2. そのフェーズの「完了条件」をすべてチェックする
3. 確認が終わったら **「確認完了しました。サーバーを停止します。」と報告してから** サーバーを停止する（`Ctrl+C`）
4. **次のフェーズには進まない。** 私の指示を待つ
5. 問題があった場合は、何が問題か・どう直すかを日本語で説明してから修正する

**重要:** 開発サーバーを起動したまま放置しない。確認が終わったら必ず停止する。
